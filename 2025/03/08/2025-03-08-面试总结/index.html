今天约了一场线上会议面试

面试过程

视频会议面试，十几分钟，有两三个问题答不上来，就结束了。

  • SpringBoot 启动的时候怎么读取的配置文件，说一下具体的流程
  • Spring 请求的执行顺序
  • Spring 缓存？
  • MySQL 优化，索引失效情况

总结

这个线上的面试给我的体验很糟糕，我不喜欢质问我的感觉，又或者说审问我而不是在交流，问完问题就噼里啪啦在打字也不知道在干什么，本身面试就是在互相选，我要想人人都满意是不可能的，答不上来感觉糟糕像是被老师提问，到时候工作了他是你领导或者组长也是这种态度，也不爽，话不投机半句多早点结束也好，有时候想想这些苦都是我自找的，想开了早点面下一个，免得影响一天的心情。

但是这两天的也有电话面试大部分是交流就没写上来，有几个公司的初试是电话技术面，电话都能聊了半个小时，都是一些遇到的场景问题，我觉得我都能答到点上，说不会也没有这么难堪也能说个大概方案出来，面试官会告诉你有什么更好的方案等等，因为问题就是让人想办法来解决的不是吗？

  • SpringBoot 启动的时候怎么读取的配置文件，说一下具体的流程

  • Spring 请求的执行顺序

  • Spring 缓存？

    他问的是 spring 自带的缓存注解，我当是说的 redis 作为一个应用用 template 引用进来。

  • MySQL 优化，索引失效情况（之前有总结过，不过答的顺序有问题他说大概知道了）

    我一上来就说联合索引的失效情况，然后还说了一些 not in is not null 这些范围不明确的条件。我答得比较乱他说知道了让我别说了，得排序一下怎么表达比较好。
